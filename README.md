# My-first-practice-
This project analyzes sales data using Python
Retail Sales Data Analysis and Performance Dashboard This project performs a comprehensive exploratory data analysis (EDA) and logical verification on a retail sales dataset of 1,000 records. Using Python, Pandas, and Plotly, it automates data cleaning, calculates missing financial metrics, and visualizes regional performance trends to drive data-informed business decisions.
**🚀 Project Overview** The primary goal of this analysis was to transform raw transactional data into actionable insights. 
The workflow includes
Data Auditing: Identifying structural issues and missingness in the dataset.
Logical Verification: Ensuring data integrity by verifying that Total_Sales consistently equals Quantity × Unit_Price.
Financial Reconstruction: Calculating missing values in critical columns like Profit and Region.
Interactive Visualization: Developing high-impact visuals to track revenue and product category performance.
**🛠️ Technical Stack Environment**
Python (via Anaconda/Jupyter Notebook). 
Data Manipulation: Pandas, NumPy. 
Visualization
Plotly Express, Plotly Graph Objects, Seaborn, Matplotlib.
File Handling
OpenpyXL (for Excel integration).
**📊 Key Insights & Visualizations**
The notebook generates several interactive and static visualizations, including:
Sales Heatmap: Analyzes the relationship between Product Categories and Geographic Regions to pinpoint high-revenue zones. Monthly Revenue Trends: A time-series analysis highlighting seasonal peaks in consumer spending. Category Performance: Bar charts comparing total sales volume across Electronics, Clothing, and Furniture.
**📂 Dataset Summary Size**: 1,000 entries across 14 columns.
Key Fields: Order_ID, Product_Category, Quantity, Unit_Price, Total_Sales, Profit, Order_Date, and Region.
Initial State: Contained approximately 4%–6% missing values across all major columns, requiring systematic cleaning.
